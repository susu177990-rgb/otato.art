# Episode Outline Template (分集大纲)

> **定位**：在进入正式分集剧本之前，为每一集产出结构化大纲，锁定本集出现的资产、叙事骨架与悬念设计。
>
> **硬规则**：
>
> 1. **逐集独立**：每集必须独立一个 `## 第N集` 块，**禁止合并集数**（如 "11-15集"），即使内容相似也必须逐集展开。
> 2. **资产标记**：∆ 行按 `∆分类：@主称谓（中文名）, ...` 格式（与设定集**同一完整串**，含全角括号与中文），须与 STAGE 5 **逐字一致**。若本集需引入新资产，须先回 STAGE 5 追加注册。
> 3. **全文 @引用**：不仅 ∆ 声明行，「开头钩子」「本集剧情」「结尾悬念」正文中**每次**提到资产也**必须**写完整 **`@主称谓（中文名）`**，裸写或半角括号或省略括注均视为格式违规。
> 4. **本集剧情须充实**：完整概括本集发生的核心事件、人物关系变化与情绪转折，不可一句话带过、不可只写标签式摘要。至少 3-5 句，让读者仅凭大纲即可理解本集故事走向。
> 5. **断行与分隔**：每集内各字段（∆场景/∆人物/∆物品/开头钩子/本集剧情/结尾悬念）之间须**各占一行**（断行），每集结束后用 `---` 水平分隔线隔开下一集，便于阅读。
>
> **与系列圣经对齐**：集数已在系列圣经中确认；大纲须覆盖全部集数。
>
> **与 STAGE 4 核心事件对齐**：每个核心事件标注了集数范围（第X集 ~ 第Y集），大纲按事件分批输出时须严格对应。
