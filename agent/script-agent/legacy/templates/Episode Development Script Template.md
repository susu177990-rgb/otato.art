# Episode Development Script Template

> **定位**：编剧室开发版分集模板。每集由 **概述区（头信息）** + **单块正文** 构成，用于 STAGE 7 **默认**输出。交付版见 `agent/script-agent/templates/Episode Final Script Template.md`（与本文不得混用于同一轮输出）。
>
> **单次使用范围（硬规则）**：**每一次** Agent 回复**只准输出「一集」**（仅一个 `## 第[集数]集`）。下一集须在主创确认后的下一轮再输出。
>
> **结构（硬规则）**：**禁止**再使用「场次 / 幕」分级；**禁止**写镜头编号、景别、机位、复杂运镜、时间段标签。正文为**一条时间线**，从头到尾写满本集。
>
> **资产标记体系（硬规则）**：
>
> - 概述区**必须**用行内 ∆ 声明本集出现的资产：`∆场景：@主名（中文名）、…`、`∆出场人物：@主名（中文名）、…`、`∆出场物品：@主名（中文名）、…`（顿号/逗号均可，**每条须为与设定集相同的完整 `@主称谓（中文名）` 串**，全角括号）。
> - **正文内每次**提到任何场景、角色、物品**必须**使用该完整串，`@` **绝对不能丢**。与 STAGE 5 设定集注册**完全一致**，一字不差。
> - 若需引入新资产，须先回到 STAGE 5 设定集追加注册。
>
> **代词（硬规则）**：正文与概述中 **禁止** 使用指向不清的代词（如：**他 / 她 / 它 / 他们 / 她们** 等）。重复写全名或 `@名称`。
>
> **台词（硬规则）**：
>
> - 台词使用 **本地化英语**，并在**同一句末尾**用括号给出 **中文翻译**：`"English line here.（中文翻译。）"`
> - **每一句台词前**须有 **括号内的情绪 / 表演 / 状态说明**，例如：`（冷着脸，声音发紧）@角色A："…"`
> - 台词仍须标明说话人：在括号说明后写 `@角色名`，再接英文引号台词。
> - **语体**：须符合编剧室侧栏 **《英语 Locale 简报》**（工程注入），**禁止中式英语（Chinglish）**；简报缺失时勿编造口语习惯，须请主创先在 Studio 生成简报。
>
> **正文书写说明（硬规则）**：
>
> - 按本集剧情发生的**先后顺序**完整书写**一段（或连贯数段）叙事**，**不拆场次、不拆幕**；不需要标注具体时间区间。
> - 写清：人物在哪、状态如何；事件如何推进；每个角色做了什么动作；说话前后与沉默时的表情与微反应；每句台词的语气与状态；关系、冲突、误会或暧昧如何变化。
> - **结尾**须用**一句台词、一个动作或一个表情**形成**集尾卡点**（在正文内写出，无需再单设「集尾」小节）。
> - **不要写**：空泛心理旁白、无效寒暄、与本集无关的解释性说明。
>
> **正文目标**：读者**只读「正文：」以下部分**，就能清楚知道本集从开始到结束发生了什么、角色怎么说与怎么演、最后卡在哪里。

## 第[集数]集：《本集标题》

本集剧情核心：（一句话概括这一集发生什么）
本集情绪走向：
时长：（秒数区间，如 60–120s）
∆场景：@…、@…
∆出场人物：@…、@…
∆出场物品：@…、@…
本集主钩子：
本集集尾卡点类型：

---

正文：

（从这里起按时间线写完整本集叙事。所有场景/人物/物品用 @名称；台词格式示例——）
（疲惫却强撑礼貌）@角色A："I said I'm fine.（我说我没事。）"
@角色B 没有接话，视线落在 @角色A 攥紧到发白的指节上。
