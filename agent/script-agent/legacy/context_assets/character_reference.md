# Reference: 角色关系与情绪驱动锚点库 (Character Relationship & Emotional Drive Anchors)

## 0. 角色库调用协议 (Asset Execution Protocol)

- **隐式运算原则**：本文件中的所有设定，仅供 Agent 在后台作为“人物引擎”进行推演。输出剧情梗概、人物小传、结构、核心事件与分集剧本时，**禁止机械复述整段角色档案**，必须将其转化为关系张力、行为选择、对白习惯与情绪后果。
- **关系优先原则**：本文件的最高用途，不是描写角色“长什么样”，而是推演角色“为什么会这样说、这样做、这样失控”。
- **动态演化原则**：角色不是静态标签。随着剧情推进，Agent 必须根据事件变化，动态更新角色的信任度、控制欲、防御强度、情绪阈值和关系站位。
- **示例隔离原则**：若仓库内附带独立示例文件，其内容仅供理解字段格式与颗粒度；若无附带文件，则以本文字段为唯一结构参考。除非主创明确指定，禁止套用任何示例角色到当前项目。示例中的姓名/国籍气质**不**构成当前项目的默认人设；海外市场项目以非华裔默认为主，除非主创另指。
- **当前项目专属原则**：每个项目都必须重新生成角色锚点。禁止把旧项目的人物关系、性格习惯、说话方式或身份结构直接挪用到新项目。

## 12. 调用准则与禁忌 (Execution Rules & Red Lines)

### 12.1 调用准则

- 梗概阶段：优先读取角色的关系驱动、核心恐惧和冲突来源。
- 人物阶段：完整建立身份、欲望、防御、触发器和对白习惯。
- 三幕式阶段：调用“必须完成的弧光”“关系权力流动”“失控阈值”。
- 核心事件阶段：调用“崩裂路径”“关系触发器”“剧情功能”。
- 分集剧本阶段：重点调用“对白习惯”“行为习惯”“亲密边界”“动态更新区”。

### 12.2 绝对禁忌

- 禁止把角色写成纯标签人设。
- 禁止只有设定，没有欲望。
- 禁止只有欲望，没有防御。
- 禁止只有拉扯，没有代价。
- 禁止让角色说出整段完整自我剖析式台词。
- 禁止前后行为逻辑断裂。
- 禁止配角只为凑人数出现。
- 禁止把本文件内容原样整段复述到正式输出里。
