# AI+实拍首帧多 Agent 工作流

这是一个独立的 AI+实拍首帧重构资源包，和 `agent/script-agent/` 平级管理。它不属于 `/chat`，也不参与现有编剧 Agent 的 prompt 加载链。

当前版本是 prompt-only 框架：只定义多 Agent 角色、输入输出契约和示例，不包含页面、API、上传、数据库、Supabase Storage、生图调用或运行时集成。

## 目标

把“原实拍首帧 + 目标场景资产宫格图 + 带位置标注的目标场景图 + 角色/道具资产 + 用户意图”整理成可执行的首帧转绘规格。

第一版只服务单帧首帧转绘，不处理视频生成、连续帧、项目版本管理或自动入库。

## Agent 分工

### AI+实拍流程主理人

文件：`agents/main-orchestrator.md`

负责流程调度、素材检查、素材索引、角色/道具绑定、调用顺序和最终结果汇总。它不直接写生图 prompt，也不直接生成图片。

### 镜头重构师

文件：`agents/shot-reconstruction-director.md`

负责判断“转绘后的首帧应该长什么样”。它输出保留原画面、废弃原画面、位置、比例、角色绑定、道具绑定、目标场景重构、关键接触关系、前中后景、禁止错误和最终首帧描述。

### 首帧转绘师

文件：`agents/first-frame-redraw-artist.md`

负责把镜头重构结果转成可执行生图规格。它必须明确使用哪些已上传素材，以及哪些图作为结构参考、场景参考、角色参考和道具参考。

## 输入素材包

页面未来应一次性上传所有素材，并整理成一个素材包。后续 Agent 只能引用素材包里的图片，不能要求用户在生图步骤重复上传。

核心输入：

- 目标场景资产宫格图
- 带位置/角色标识的目标场景资产宫格图
- 目标角色图列表，每个角色必须绑定角色名称
- 目标道具图列表，每个道具必须绑定道具名称，可选绑定关联角色
- 原实拍视频片段首帧图
- 用户文字意图/注意事项（可选；不填时由 Agent 根据图片和标注自行分析）

## 输出阶段

1. 主理人输出素材检查和调度包。
2. 镜头重构师输出结构化镜头重构方案。
3. 首帧转绘师输出可执行生图规格。
4. 未来运行时再把生图规格交给项目已有 image generation API。

## 资源文件

- `manifest.json`：工作流登记信息。
- `schemas/asset-bundle.schema.json`：素材包结构草案。
- `schemas/reconstruction-output.schema.json`：镜头重构输出结构草案。
- `schemas/redraw-output.schema.json`：首帧转绘输出结构草案。
- `examples/yacht-case.md`：游艇案例的文本示例。
