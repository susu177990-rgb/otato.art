# oTATo Art

oTATo Art 是一套面向内容创作团队的统一工作台，把项目立项、编剧推进、视觉生成、素材整理和创作记录收进同一个产品里。它不是把一堆工具塞进同一个页面，而是把原本分散在聊天窗口、文档、表格、生图站和素材库里的工作，整理成一条连续、可回看、可继续推进的创作链路。

如果你在做短剧、系列化影视、流媒体内容、角色资产、分镜视觉或改编前期，oTATo Art 解决的不是单点能力，而是整个创作过程里的上下文断裂问题。

## 为什么它有价值

- 你不用再在“找素材、立项目、写剧本、出视觉、查历史”之间来回切换工具。
- 同一个项目里的上下文可以连续保留，不会因为换了页面、模型或流程就丢掉创作状态。
- 视觉生成、预设、会话、项目和画布不是孤立功能，而是围绕同一套工作流协同运转。
- 整体界面已经统一为生产化线框风，更接近一个真正可长期使用的创作系统，而不是拼接起来的功能集合。

## 适合谁

- 做短剧、影视、系列内容策划的人
- 需要从改编素材快速进入立项和写作的人
- 需要一边写文本、一边做角色和视觉参考的人
- 希望把 AI 对话、项目资料、画布组织和生成结果收进同一空间的团队

## 你可以在里面完成什么

### 1. 从想法到项目

你可以先建立项目，再进入 onboarding，把基本信息、参考资料、策划讨论和立项确认收在同一个流程里。它更像一个真正的创作起步空间，而不是一张需要自己维护逻辑的表格。

### 2. 从项目到写作

进入编剧工作区后，项目不再只是存档，而是继续向前推进。你可以围绕阶段化写作、对话协作、结构化产物和历史记录持续工作，不需要把创作过程拆散到多个工具里。

### 3. 从文本到视觉

生图、生视频、对话和无限画布不是独立页面堆在一起，而是共享同一套设计和操作语言。你可以在独立工作台里生成，也可以在无限画布里继续组织、拼接、比较和延展。

### 4. 从生成到沉淀

生成结果不是一次性内容。项目、会话、画廊、预设、历史记录和画布节点都可以成为后续继续工作的基础，这一点对长期内容生产尤其重要。

## 相比常见做法，它的优势是什么

很多创作型产品只解决单个动作，比如只会聊天、只会生图、只会存文档，最后还是要靠用户自己在外部把流程拼起来。oTATo Art 的优势在于它把整条创作链路产品化了：

- 首页就是工作入口，不是展示页
- 项目、对话、视觉、画布之间可以连续切换
- 预设、模型、历史、参考图和结果都属于工作流的一部分
- 不是为了“炫 AI”，而是为了让创作过程更可控、更可追踪、更容易复用

## 核心体验

### 项目工作台

围绕项目推进内容创作，适合把零散的创意、资料和决策收束成可以继续执行的状态。

### 对话工作台

适合连续对话、技能表单输入、会话管理和上下文延续，不只是一个普通聊天页。

### 生图与生视频工作台

统一的操作栏、预设逻辑、参考图处理和历史结构，让生成过程更像专业工具，而不是一次性的 prompt 提交器。

### 无限画布

把文本、素材、预设、生成结果和关系组织到同一个空间里，适合整理思路、搭建视觉方向和继续推进复杂项目。

## Features

- 一个项目里同时容纳文本创作和视觉创作
- 工作流连续，不需要频繁跳出到外部工具补流程
- 预设、历史、画布、生成结果都能继续复用
- 界面已经按统一设计合同重整，风格一致，适合长期迭代
- 保留现有功能逻辑，不为了改界面牺牲原本能工作的流程

## Installation

### 安装

```bash
npm install
```

### 配置

在仓库根目录创建 `.env.local`，至少提供 Supabase 所需变量：

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
APP_ORIGIN=http://localhost:4000
```

应用内的模型、网关、预设和相关工作区设置，统一在站内 `设置` 页面维护。

### 启动

```bash
npm run dev
```

打开 `http://localhost:4000`。

如果你只想启动 Web 前端：

```bash
npm run dev:web
```

## Usage

最典型的进入方式是：

1. 进入首页，选择当前要做的工作模式
2. 创建或进入一个项目
3. 在对话、生成、画布之间来回推进，而不是把它们当成彼此无关的页面
4. 通过历史、画廊、预设和项目状态继续下一轮创作
